# MVP 인수 기준

아래가 모두 충족되어야 “완료”다.

## A. 저장소와 실행

- [ ] `pnpm install` 후 문서화된 명령으로 실행된다.
- [ ] `.env.example`이 모든 환경변수를 설명한다.
- [ ] API key 없이 mock mode로 전체 흐름이 동작한다.
- [ ] PostgreSQL은 Docker Compose로 로컬 실행 가능하다.
- [ ] migration과 seed 명령이 있다.

## B. 캐릭터 조회

- [ ] 유효한 이름을 검색하면 normalized profile을 반환한다.
- [ ] live mode에서 key는 server-only다.
- [ ] live provider가 `id`, `character/basic`, `character/stat`, `character/item-equipment`을 실제 공식 경로와 Zod 응답 검증으로 조회한다.
- [ ] 전투력은 `character/stat.final_stat`에 API가 제공한 원값일 때만 표시하며, 서비스 계산값이나 기본값으로 대체하지 않는다.
- [ ] 현재 장착 전투 장비 전체의 슬롯, 아이템명, 옵션, 스타포스, 잠재능력, 에디셔널 잠재능력, 소울을 normalized profile에 보존한다.
- [ ] 심볼·캐시 장비·세트 효과 보강 호출은 별도 모델에 보존하고, 캐시 장비를 전투 장비와 혼동하지 않는다.
- [ ] 이름이 없거나 너무 길거나 잘못된 문자는 validation 오류다.
- [ ] 일부 optional endpoint 실패 시 기본 프로필로 계속 진행할 수 있다.
- [ ] typed error가 한국어 메시지로 변환된다.
- [ ] mock fixture가 정상/부분 누락/stale 사례를 포함한다.

## C. 편집기

- [ ] 주간/월간 구분과 난이도가 포함된 목표 보스, 역할, 파티 유형을 입력할 수 있다.
- [ ] 주간·월간 선택 카드는 키보드로 조작 가능하며, 사용자가 사용을 허가한 Maple Trackers 원본 아트 또는 offline fallback만 사용한다.
- [ ] 요일과 시간대를 1개 이상 입력할 수 있다.
- [ ] 경험, 역할 설명, 음성 채팅, 분배 방식, 선택 연락 방법을 입력할 수 있다.
- [ ] 모든 입력은 Zod로 server-side 재검증한다.
- [ ] 입력 길이와 허용값을 제한한다.
- [ ] 실시간 미리보기는 모바일/데스크톱에서 사용 가능하다.

## D. 출처와 내용

- [ ] API 조회/서비스 계산/작성 내용/서비스 관측 배지가 존재한다.
- [ ] 작성 내용이 API 값과 같은 스타일로 오인되지 않는다.
- [ ] 누락 값은 0으로 위조되지 않는다.
- [ ] 메숭이 체온 외의 단일 종합 점수와 합격 판정이 없다.
- [ ] 역할에 따라 핵심 필드 우선순위가 달라진다.
- [ ] 현재 지원 범위가 아닌 기록 지표를 공개 카드·검증 페이지·공유 이미지에 노출하지 않는다.

## E. 게시와 권한

- [ ] 게시 시 unique public slug와 edit token이 생성된다.
- [ ] edit token 원문은 DB에 저장되지 않는다.
- [ ] token 없는 사용자는 수정·삭제·갱신할 수 없다.
- [ ] 작성자는 같은 브라우저에서 갱신/수정/공개 중단할 수 있다.
- [ ] 공개 중단 상태는 적절한 404/410 또는 비공개 화면을 반환한다.

## F. 검증 페이지

- [ ] `/r/[slug]`가 snapshot, 작성 내용, 출처를 표시한다.
- [ ] 데이터 기준 시각과 source date를 표시한다.
- [ ] fresh/stale/expired 상태를 표시한다.
- [ ] 최신 version이 있으면 오래된 version에 안내한다.
- [ ] 연락 방법은 작성자가 공개를 선택한 경우에만 표시한다.
- [ ] API 조회 전투력과 현재 장착 전투 장비 전체를 출처 라벨과 함께 확인할 수 있다.
- [ ] 장비 상세는 옵션·스타포스·잠재·에디셔널 잠재·소울을 보존하고, 심볼·캐시 장비·세트 효과는 전투 장비와 구분한다.
- [ ] NEXON 데이터 및 비제휴 고지가 있다.

## G. 이미지

- [ ] 1080×1350 PNG가 서버에서 생성된다.
- [ ] 캐릭터명, 직업, 레벨, 목표 보스, 역할, 핵심 지표, 일정이 읽힌다.
- [ ] 데이터 기준 시각, version, 출처 범례가 있다.
- [ ] QR이 canonical 공개 페이지를 가리킨다.
- [ ] 이미지와 페이지가 같은 ResumeVersion/snapshot을 사용한다.
- [ ] 이미지에는 장비 전체 옵션을 넣지 않고, 전투력과 역할별 핵심 API 지표만 요약한다.
- [ ] 외부 캐릭터 이미지 실패 시 placeholder로 렌더링된다.
- [ ] API key/edit token/비공개 연락처가 포함되지 않는다.

## H. 신선도와 길드 관측

- [ ] 24시간/30일 기본 정책이 unit test로 검증된다.
- [ ] 갱신은 기존 version을 덮어쓰지 않고 새 version을 만든다.
- [ ] 길드 변경 시 observation transition이 테스트된다.
- [ ] 실제 가입/탈퇴일이 아니라 관측 기간으로 표현된다.

## I. 품질

- [ ] `pnpm lint` 통과
- [ ] `pnpm typecheck` 통과
- [ ] `pnpm test` 통과
- [ ] `pnpm test:e2e` 통과
- [ ] `pnpm build` 통과
- [ ] 핵심 흐름이 375px viewport에서 사용 가능하다.
- [ ] 능력치·장비 정규화와 주간/월간 보스 선택의 핵심 흐름이 unit 또는 E2E 테스트로 검증된다.
- [ ] 폼 label, focus, keyboard navigation, 오류 연결이 존재한다.
- [ ] README에 mock/live/production 설정과 한계가 정리되어 있다.

## J. 메숭이 체온 익명 설문

- [x] 기본 36.5℃에서 보스 경험·숙련도(-2~+2), 시간 약속(-1/+1)의 설문 합계를 누적한다.
- [x] 설문 링크는 작성자의 edit token으로만 발급되고 7일 뒤 만료되며, 원문 token은 저장하지 않는다.
- [x] 설문 제출은 메력서, 닉네임, 연락처, reviewer OCID를 요구하거나 저장하지 않는다.
- [x] 같은 invitation의 재사용, 만료, 다른 캐릭터 이력서 대상 제출을 차단한다.
- [x] 캐릭터 OCID 단위로 누적한 온도는 새 Resume·version에서도 유지되고, 0.0~100.0℃ 범위로 제한된다.
- [x] 공개 검증 페이지는 현재 온도·게이지·익명 응답 수만 보여 주며, 설문자 신원·개별 답변·자유 댓글·순위는 보여 주지 않는다.
- [x] 메숭이 체온은 1080×1350 PNG, QR 공유 이미지, 이력서 글 복사 및 합격/매칭 판단에 포함하지 않는다.
