# 메력부 제품 명세

## 1. 제품 개요

### 서비스명

- 서비스: **메력부 — 메이플 이력부**
- 결과물: **메력서 — 메이플 이력서**
- 슬로건: **스펙을 다시 쓰지 말고, 한 장으로 증명하세요.**

### 한 줄 정의

캐릭터명을 검색하고 지원할 보스·역할·가능 시간을 선택하면, NEXON Open API 데이터를 기반으로 파티 구직용 캐릭터 이력서 이미지와 검증 프로필 URL을 생성한다.

### 제품 포지셔닝

메력부는 이미지 꾸미기 도구가 아니다. 이미지에 담긴 수치의 출처, 데이터 기준 시각, 최신 버전을 확인할 수 있는 **검증 가능한 캐릭터 프로필 서비스**다.

## 2. 해결할 문제

파티 지원자는 캐릭터명, 직업, 레벨, 전투 관련 수치, 유니온, HEXA, 장비, 보스 경험, 가능한 시간, 음성 채팅, 분배 방식을 반복해서 작성한다. 파티장은 사람마다 다른 표기와 단위, 오래된 스크린샷, 직접 입력한 수치를 비교해야 한다.

메력부는 다음으로 해결한다.

1. 조회 가능한 정보는 API 자동 입력
2. 공식 API가 아닌 계산값은 공식/버전/기준일 표시
3. 경험·일정 등은 사용자 입력으로 명확히 구분
4. 한 장의 표준 카드와 공개 검증 페이지 제공
5. 이미지가 편집되거나 오래되어도 QR/URL에서 원본과 최신 버전 확인

## 3. 핵심 사용자

### 파티 지원자

- 정보를 반복 작성하지 않고 빠르게 지원하고 싶다.
- 자신의 역할, 가능한 시간, 경험을 스펙과 함께 보여 주고 싶다.

### 파티장

- 지원자를 같은 기준으로 빠르게 비교하고 싶다.
- 이미지 수치가 언제 조회됐고 어디서 왔는지 확인하고 싶다.

### 2차 사용자

- 길드 운영진
- 구직 게시판/커뮤니티 운영자

MVP의 제품 결정은 파티 지원자와 파티장 흐름을 우선한다.

## 4. 사용자 흐름

### 지원자

1. 랜딩에서 캐릭터명 검색
2. API/mock 결과 확인
3. 주간/월간 구분과 목표 보스, 난이도, 역할, 파티 유형 선택
4. 가능 시간, 음성 채팅, 분배 방식, 보스 경험 입력
5. 메력서 미리보기 확인
6. 게시하여 공개 URL과 이미지 생성
7. 이미지 또는 URL 공유
8. 나중에 최신 데이터로 갱신하여 새 version 생성

### 파티장

1. 채팅방에서 메력서 이미지 확인
2. 지원 분야와 핵심 지표를 빠르게 확인
3. QR/URL로 검증 페이지 이동
4. 데이터 기준 시각, 출처, 상세 정보, 최신 version 확인
5. 공개된 연락 방법으로 지원자에게 연락

## 5. 정보 출처

| 출처 | 표시 | 예시 |
|---|---|---|
| NEXON API | API 조회 | 레벨, 직업, 종합 능력치의 전투력 원값, 현재 장착 전투 장비, 현재 길드 |
| 서비스 계산 | 서비스 계산 | 버전이 있는 파생 지표 |
| 사용자 입력 | 사용자 입력 | 경험, 일정, 음성 채팅, 분배 방식 |
| 서비스 관측 | 서비스 관측 | 서비스가 여러 시점에 발견한 길드 변경 |

출처는 field 또는 section 수준으로 보존한다. API 조회는 제휴/보증/숙련 인증을 의미하지 않는다.

## 6. 메력서 정보 구조

### 프로필

- 캐릭터 이미지
- 캐릭터명
- 월드
- 직업
- 레벨
- 현재 길드
- 데이터 기준 시각

### 지원 분야

- 보스 구분: 주간/월간
- 목표 보스
- 난이도
- 역할: 격수/보조/지원/기타
- 파티 유형: 고정/반고정/임시/트라이

주간·월간 선택 카드는 보스 종류를 빠르게 구분하는 UI다. 카드에 쓰는 일러스트는 자체 생성한 추상 판타지 아트이며, 특정 게임 캐릭터·공식 보스 초상·공식 제휴를 암시하지 않는다. 실제 지원 대상 보스명과 난이도는 사용자 입력으로 보존하고, API 조회값처럼 표시하지 않는다.

### 핵심 역량

API에서 신뢰성 있게 제공되는 항목 중 역할에 맞는 4~6개만 강조한다. `/character/stat`의 `final_stat`에 포함된 전투력은 API 원값일 때만 표시하며, 서비스 계산·환산·등급으로 바꾸지 않는다. 항목명이 불명확하거나 누락된 값은 억지로 표시하지 않는다.

### 성장 및 장비

- 유니온
- HEXA 요약
- 현재 장착 전투 장비 요약
- 공개 검증 페이지의 장비 상세: 장착 슬롯, 아이템명, API 제공 옵션, 스타포스, 잠재능력, 에디셔널 잠재능력, 소울, 심볼, 세트 효과

장비 상세는 검증 페이지에서만 펼쳐 보여 준다. 1080×1350 공유 이미지는 핵심 지표를 우선하므로 모든 장비 옵션을 넣지 않는다. 캐시 장비는 전투 장비와 분리해 선택적으로 표시한다.

### 파티 경력

- 보스 경험
- 어필 포인트
- 고정 파티 경험 기간
- 숙련도 설명

초기에는 사용자 입력이며 “인증” 문구를 쓰지 않는다.

### 소속 이력

- 현재 길드: API 조회
- 서비스가 관측한 과거 길드: 서비스 관측
- 사용자가 추가한 과거 이력: 사용자 입력

### 희망 조건

- 가능한 요일/시간
- 시간대
- 음성 채팅 가능 여부
- 분배 방식
- 간단한 소개
- 선택 연락 방법

### 하단 검증

- snapshot 기준 시각
- 메력서 version
- content hash 일부
- QR
- 짧은 공개 URL
- 출처 범례
- “Data based on NEXON Open API”
- “본 서비스는 NEXON의 공식 제휴/인증 서비스가 아닙니다.”

## 7. 역할별 표시 전략

### 일반 격수

주요 공격/보스 관련 API 지표, 전투력 원값, 유니온, HEXA를 우선한다.

### 보조/지원 역할

파티 기여와 관련된 공개 가능한 정보, 주요 세팅, 지원 가능 시간과 역할 설명을 우선한다. 격수용 숫자를 그대로 합격 기준처럼 사용하지 않는다.

### 트라이 파티

일정 안정성, 학습 경험, 소통 방식, 트라이 가능 시간을 강조한다.

### 숙련 파티

클리어 경험, 어필 포인트, 일정 안정성을 강조한다.

직업별/역할별 단일 점수는 만들지 않는다.

## 8. MVP 기능 우선순위

### P0

- 캐릭터 검색 및 기본 조회
- API/mock normalized profile
- 지원 조건 편집
- 실시간 카드 미리보기
- 공유 이미지
- 공개 검증 페이지
- QR/URL
- 출처 라벨
- 기준 시각과 stale 상태
- 편집 권한 token
- demo 모드

### P1

- 여러 메력서 저장
- 최신 API 데이터 갱신 및 version 비교
- 테마 2~3종
- 로그인/계정 연결
- 공개 범위 세분화

### P2

- 길드 관측 이력
- 성장 변화 그래프
- 파티장 지원자 관리 보드
- 커뮤니티/Discord 연동
- 검증 가능한 보스 경험 방식 연구

## 9. 비목표

- 합격 점수 또는 티어 자동 판정
- 익명 리뷰/평판
- 블랙리스트
- 자동 파티 매칭
- 대리 구직/거래
- API에서 제공되지 않는 보스 클리어 이력의 허위 자동 인증
- 최초 방문 캐릭터의 전체 과거 길드 이력 재구성

## 10. 성공 지표

핵심 funnel:

`캐릭터 검색 → 조회 성공 → 작성 완료 → 게시 → 이미지/링크 공유 → 검증 페이지 조회 → 연락 행동`

보조 지표:

- 검색 대비 게시 완료율
- 게시물 공유율
- 이미지 대비 검증 URL 조회율
- 재갱신율
- API 실패율
- 이미지 생성 성공률/시간
- stale/expired 카드 비율
- 신고율
- 파티장 후보 판단 시간

## 11. 수익 모델 방향

핵심 검증 기능은 무료로 유지한다.

유료 후보:

- 프리미엄 테마
- 보스별 메력서 세트
- 자동 갱신/성장 리포트
- 장기 관측 이력
- 파티장 지원자 보드
- 길드/커뮤니티 브랜딩

결제 여부로 데이터 신뢰도나 노출 순위를 차등하지 않는다.

## 12. 주요 위험과 대응

| 위험 | 대응 |
|---|---|
| 이미지 조작 | QR, 원본 URL, version, content hash |
| 오래된 데이터 | 기준 시각, stale/expired, 갱신 흐름 |
| 계산식 변경 | algorithm version과 input snapshot 저장 |
| 직업 편향 | 역할별 우선순위, 종합 점수 금지 |
| 스펙이 숙련도를 대체 | 경험·일정·소통 항목을 동등하게 노출 |
| 길드 과거 정보 부족 | 관측 이후만 검증 가능한 이력으로 표시 |
| API 장애/변경 | adapter, cache, mock, schema validation |
| 사칭/괴롭힘 | 선택 공개, edit token, 신고/비공개 상태 |
| 정보 과밀 | 이미지 요약, 웹 상세 분리 |
